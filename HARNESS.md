# Regulation Site HARNESS (Revised)

## 목적

이 프로젝트는 건축 법규 자동화 웹사이트이다.
핵심 목표는 "법규를 자동으로 대신 판단하는 것"이 아니라,
주소/필지 기반 정보와 계획 용도를 바탕으로
건축 관련 법규를 빠르게 연결하고,
실무자가 **법규 검토 흐름을 끊지 않고 다음 단계로 넘어갈 수 있게 돕는 것**이다. 

또한 본 시스템은
**자동 판정 + 수동 검토 안내 + 조례 탐색 + 보고서 출력**까지
하나의 흐름으로 제공하는 실무형 검토 도구를 목표로 한다.

---

## 전체 플로우 (고정 + 확장)

[1] 주소 입력
[2] 필지 식별
[3] 토지 법규 조회
[4] 용도군 출력
[5] 계획 용도 선택
[6] 건축 + 관련 법규 전체 묶음 출력
[7] 우선 검토 항목 출력
[8] 주의사항 출력
[9] 계산 + 판정 통합 출력
[10] 수동 검토 / 조례 확인 / 법규 탐색
[11] 검토 결과 저장 및 보고서 출력

이 순서는 프로젝트의 핵심 UX이므로,
사용자 요청 없이 임의로 합치거나 생략하거나 재구성하지 않는다.

---

## 이 시스템의 본질

이 시스템은 "법을 계산하는 시스템"이 아니다.
이 시스템은 "법규를 빠르게 연결하고 검토 흐름을 구조화하는 실무 도구"이다.

---

## 입력 데이터 구조 (고정)

### rawInputs (사용자 입력)

* 대지면적
* 건축면적
* 연면적
* 층수
* 높이
* 도로폭
* 용도 관련 상세 입력

### derivedInputs (서버 계산)

* 건폐율
* 용적률
* 주차 요구량
* 피난/소방 트리거

### planningContext

* 용도지역
* 지구/구역
* overlay 정보
* 도시계획시설
* 개발행위허가 여부

### manualReviewContext

* 조례 확인 필요 여부
* 지구단위계획 문서 필요 여부
* 협의 필요 항목

---

## 자동 판정 / 수동 검토 구분 (필수)

### 자동 판정 가능

* 용도 허용 여부 (1차)
* 건폐율 / 용적률
* 일부 높이 기준
* 기본 주차 기준
* 일부 피난 트리거

### 기준 참고

* 입력 부족 시 기준만 제시

### 수동 검토

* 조례
* 지구단위계획 문서 해석
* 소방 / 설비 / 전기
* 일조 / 사선 / 이격거리
* 협의 필요 항목

---

## 법규 레이어 구조 (반드시 유지)

### 레이어 1. 건축 기본 법규 (Core)

* 건축법
* 건축법 시행령
* 건축법 시행규칙
* 국토계획법
* 주차장법
* 장애인 관련 법

---

### 레이어 2. 건축 필수 연계 법규 (Extended Core)

* 에너지
* 피난
* 방화
* 채광 / 환기
* 위생

---

### 레이어 3. 협력사 법규 (MEP)

* 소방
* 설비
* 전기
* 승강기

원칙

* 자동 판정 금지
* 반드시 “검토 필요”로 출력

---

## 조례 레이어 (신규 추가)

조례는 자동 판정하지 않는다.

### 출력 구조

* “조례 확인 필요” 카드
* 관련 조례 카테고리
* 검색 키워드
* 지자체 링크

### 원칙

* 지역별 상이 → 반드시 수동 확인
* 자동 판정 금지
* 누락 방지 목적

## 최종 판단 기준

이 시스템은
“법규를 대신 판단하는 시스템”이 아니라

**건축사가 빠르게 검토하고,
놓치지 않고,
다음 단계로 넘어갈 수 있게 만드는 시스템**이다.
